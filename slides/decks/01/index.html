Introduction

Node.js

What is node.js?

  • Node.js is a JavaScript runtime
  • An event-driven architecture with a non-blocking I/O API

Why use node.js

  • JavaScript across all layers (frontend and backend)
  • Scalable (no overhead from OS processes and/or thread handling)
  • Simplified programming model (combines the ease of a scripting language with the power of Unix network programming)

Architecture

  • In Node.js there are two types of threads:
    • One Event Loop (aka the main loop, main thread, event thread, etc.)
    • A pool of k Workers in a Worker Pool (aka the threadpool)
  • The Event Loop are responsible for:
    • Executing JavaScript callbacks registered for events
    • Fulfilling non-blocking asynchronous requests
  • The Worker Pool handles expensive tasks:
    • I/O-intensive tasks like DNS and File System
    • CPU-intensive tasks lik crypto and compression
